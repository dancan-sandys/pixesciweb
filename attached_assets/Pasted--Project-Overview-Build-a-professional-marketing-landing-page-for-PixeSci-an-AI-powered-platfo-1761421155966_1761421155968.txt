  Project Overview

  Build a professional marketing landing page for PixeSci - an AI-powered platform that automates scientific software through natural language commands. This is a
  marketing-focused site designed to capture user interest and waitlist signups, NOT a business pitch or revenue-focused page.

  Target Audience

  - PhD students and postdocs struggling with complex software
  - Core facility managers handling repetitive analysis requests
  - Research scientists who want to focus on science, not software
  - Lab technicians who need to use expensive tools they weren't trained on
  - PIs and lab directors wanting to standardize workflows across their teams

  Brand Colors & Theme

  Use the PixeSci app color scheme:
  - Primary: Deep scientific blue (#1e3a8a)
  - Secondary: Bright cyan/teal (#06b6d4)
  - Accent: Electric blue (#3b82f6)
  - Background: Clean white (#ffffff) with subtle gray sections (#f8fafc)
  - Text: Dark gray (#1f2937) for body, white for CTAs
  - Success/CTA: Vibrant green (#10b981)

  Visual Style: Modern, clean, scientific aesthetic with subtle gradients and professional imagery.

  Technical Stack

  Build using:
  - React + Vite (for fast development)
  - Tailwind CSS (for rapid styling with the color scheme above)
  - Framer Motion (for smooth animations)
  - React Hook Form (for waitlist form)
  - Responsive design (mobile-first approach)

  Animation & Interaction Guidelines

  Micro-Animations:

  - Hover effects on all buttons and cards
  - Smooth scrolling between sections
  - Fade-in animations as elements enter viewport
  - Tool logos with gentle hover animations
  - Progress indicators for the demo section

  Performance:

  - Lazy loading for images
  - Optimized animations (60fps)
  - Fast loading (<2 seconds)
  - Mobile responsive (all content works on phone)

  Form Integration

  Set up the waitlist form to:
  - Collect emails in a database/service
  - Send confirmation emails
  - Track signup source
  - Export data for follow-up campaigns
  - Add to email marketing platform

  Call-to-Action Strategy

  Primary CTA: "Join the Waitlist" (used 3-4 times)
  Secondary CTAs: "Watch Demo", "Learn More"
  CTA Placement: Hero, after problem section, after demo, final footer

  Success Metrics

  The site should optimize for:
  - Email signups (primary goal)
  - Time on page (engagement)
  - Demo completion (interest level)
  - Mobile conversion (60% of traffic)

  ---
  Build this as a single-page application with smooth scrolling navigation. Focus on emotional connection - scientists frustrated with software complexity who just want to do 
  their research. The tone should be empathetic, excited, and confident without being overly technical or salesy.
